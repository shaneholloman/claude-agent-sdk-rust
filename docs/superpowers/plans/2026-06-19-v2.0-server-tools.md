# v2.0 Server Tools & Type Overhaul Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the Tool type system to support server-side tools (web search, code execution, bash, text editor), add their corresponding content blocks, and update related types (ToolChoice, ThinkingConfig, OutputConfig, ToolResult).

**Architecture:** The current `Tool` struct becomes `ToolDefinition`, a tagged enum with `Custom` (existing behavior) plus server tool variants. `ContentBlock` gains new variants for server tool results. `ToolChoice` absorbs `disable_parallel_tool_use`. `ThinkingConfig` gains `Adaptive` variant. `OutputConfig` gains JSON schema format. A new `count_tokens` API endpoint is added to `ClaudeClient`.

**Tech Stack:** Rust 1.70+, serde (with custom Serialize/Deserialize), reqwest, tokio

**Breaking changes in this release:**
- `Tool` renamed to `CustomTool`, new `ToolDefinition` enum replaces it in `MessagesRequest.tools`
- `ToolChoice` variants gain struct fields
- `MessagesRequest.disable_parallel_tool_use` removed
- `ThinkingConfig::Enabled` gains `display` field
- `ToolResult.content` type changes from `Option<String>` to union type
- `ToolUse` gains `caller` field

**Files:**

Rename the existing `Tool` struct to `CustomTool`. Create a new `ToolDefinition` enum that wraps it plus all server tool types. This is the core breaking change.

- [ ] **Step 1: Rename `Tool` to `CustomTool`**

Keep the struct identical, just rename. Add a type alias `pub type Tool = CustomTool;` for backward compat (deprecated).

- [ ] **Step 2: Create `ToolDefinition` enum**

```rust
/// Tool definition — either a custom client tool or a built-in server tool.
///
/// This is the type used in `MessagesRequest.tools`.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(untagged)]
pub enum ToolDefinition {
    /// Custom client-side tool with name, description, and JSON schema
    Custom(CustomTool),
    /// Built-in web search
    WebSearch(server_tools::WebSearchTool),
    /// Built-in web fetch
    WebFetch(server_tools::WebFetchTool),
    /// Built-in code execution (sandboxed Python)
    CodeExecution(server_tools::CodeExecutionTool),
    /// Built-in bash execution
    Bash(server_tools::BashTool),
    /// Built-in text editor
    TextEditor(server_tools::TextEditorTool),
}
```

Note: Uses `#[serde(untagged)]` because each variant already has its own `type` field with different values — serde will try each variant in order.

- [ ] **Step 3: Change `MessagesRequest.tools` from `Option<Vec<Tool>>` to `Option<Vec<ToolDefinition>>`**
